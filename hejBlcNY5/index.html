6. Git--分支

Posted on 2022-02-12，1 min read

6. 分支

线上代码出现BUG进行修复（对出现BUG的代码进行修改并合并到主分支）

6.1 master(默认)

主枝干，线上运行的代码，开发主线

6.2 分支

每个分支都是隔离的环境，互不影响。所以分支的作用是隔离环境。

6.3 命令

git branch # 查看分支
git branch branch_name # 创建分支
git checkout branch_name # 切换分支

# 将分支合并到master
# 1.切换回master
git checkout master
# 2.合并
git merge branch_name
# 3.删除分支
git branch -d branch_name

6.4 冲突

# 当将分支合并到master时，产生冲突，需要手动修改文件中的冲突，然后提交
# 1.修改
# 2.add
# 3.commit
